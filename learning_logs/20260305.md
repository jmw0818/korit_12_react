## 🟣 React List
```jsx
const arr = [1,2,3,4,5];
const resArr = arr.map(x => x*2);
// arr배열의 내부 elemnt를 임의로 x로 선언하고 *2를 수행했다.
```
```jsx
export default function MyList() {
  const data = [1,2,3,4,5];
  return (
    <ul>
      {
        data.map((elem, index) =>
          <li key={index}>List Item : {elem*2}</li>
        )
      }
    </ul>
  );
}
```
- 처음엔 `data.map(elem => <li>List Item : {elem*2}</li>)`으로 작성했다. (개발자 도구에서 Each child in a list should have a unique "key" prop. 이라는 경고문이 떠서 index를 추가하였음 -> 리액트 list 항목에는 행이 업데이트, 추가, 삭제 되었는지를 감지하는 데 이용되는 고유 키(unique key)가 필요하다. .map() 메서드에서는 두 번째 argument로 index가 존재하는데, 이는 경고 처리를 하는데 이용된다.)

- 다만 index를 추가하는 방식은 임시방편이라고 볼 수 있다. 목록을 추가, 삭제, 혹은 정렬을 할 경우 에러 가능성이 존재하기 때문에 index를 key로 사용하는 것은 적절하지 않다. 데이터의 고유 키가 존재한다면 그것을 이용하는 것이 좋다. (DB에 저장된 각 테이블의 PK) 그런데 엔티티의 필드 목록을 드러내기 싫다면 프론트에서만 사용하는 임의적인 고유 키가 있어야하는데, 이는 uuid 개념을 검색하고 학습하면 된다.

- element가 객체인 배열이라면, 단순 list로 풀어내기 보다는 테이블 형태로 풀어내는 것이 더 적절하다. 속성들이 header/column 명이 되는 방식으로 내부 값들을 보여주면 되겠다. 해당 경우 `<li>`보다는 `<tr>/<td>`를 사용하는 것이 좋다. 

```jsx
export default function MyTable() {
  const data = [
    {id: 1, brand: '현대', model: '그랜저'},
    {id: 2, brand: '기아', model: '셀토스'},
    {id: 3, brand: '람보르기니', model: '우라칸'}
  ];

  return(
    <>
      <table>
        <tbody>
          {
            data.map(car => 
              <tr key={car.id}>
                <td>{car.brand}</td>
                <td>{car.model}</td>
              </tr>
            )
          }
        </tbody>
      </table>
    </>
  );
}
```
- 주의할 점은 arrow function시에 사람이 읽을 경우 여러줄 이지만 컴퓨터가 읽었을 때는 다수의 html 태그가 한 줄로 나열되어있는 것이기 때문에 구현부를 나타내는 { }가 없다.

## 🟣 React Event
- 리액트에서의 이벤트처리는 DOM 요소 이벤트 처리와 유사하다. 클릭 했을 때 ul에서 li하나를 추가시켜서 읽어왔었다. (todolist) html 상에서의 DOM요소 처리는 태그 내에 소문자(onclick)이지만, React 상에서는 camel case를 이용한다.

```jsx
export default function MyComponent() {
  const handClick = () => {
    alert('Button Pressed !');
  }

  return(
    <>
      <button onClick={handClick}>Press Me</button>
    </>
  );
}
```
- 이상의 코드에서 주의해야 할 점은 `<button onClick={handClick()}>Press Me</button>`로 작성하면 안된다. `handClick()` 소괄호 유무가 앱 실행시(렌더링시)에서의 차이점이 생긴다. ()를 포함했을 경우에는 버튼을 누르기 전에 이미 alert 창이 뜬다는 차이점이 있다.

## 🟣 React Form
```jsx
import { useState } from "react";

export default function MyForm2() {
  const [ text, setText ] = useState('');

  const handleChange = evnet => {
    console.log(evnet.target.value);
    setText(evnet.target.value);
  }

  const handleSubmit = event => {
    alert(`당신은 ${text} 라고 입력하셨네요 !`);
    event.preventDefault();
  }
  return(
    <>
      <form action="" onSubmit={handleSubmit}>
        <input type="text" onChange={handleChange} value={text} placeholder="내용을 입력하세요." />
        <input type="submit" value="Press Me!" />
      </form>
    </>
  );
}
```
- 이상에서 Form처리라고 하긴 했지만 event 처리 관련 하나 더 학습했다. 자주 사용하는 input에서의 onChange 이벤트인데 defualt 형태이다보니 별개의 함수로 작성하지 않고 inline으로 작성하기도 한다.
`onChange={event => setText(event.target.value)}`

- 이상에서까지는 기본적으로 form 태그 내에 input이 하나 밖에 없었지만, 회원가입을 하든 로그인을 하든 todolist가 아니라면 웬만하면 input field가 복수로 있을 것이다.
```jsx
const [ username, setUsername ] = useState('');
const [ password, setPassword ] = useState('');
... 중략

<input type="text" onChange={event => setUsername(event.target.value)}>
<input type="password" onChange={event => setPassword(event.target.value)}>
```
와 같이 작성하게 될 것이다. 그렇다면 횟수만큼 익명함수를 하나하나 작성해야하는데, setter 개수가 field수 만큼 있다보니 실수가 발생할 수도 있을 것이다. 이를 처리하기 위해서 useState()의 initialValue를 JS 객체로 선언하는 방법도 있겠다.
